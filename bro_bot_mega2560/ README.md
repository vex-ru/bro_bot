# 🤖 bro_bot_mega2560

Низкоуровневый драйвер шасси для робота **bro_bot** на базе Arduino Mega2560 и VexV5 Shield. Обеспечивает управление дифференциальным приводом, расчёт одометрии в реальном времени и текстовый командный интерфейс для связи с上位控制系统 (ESP32/MicroROS).

---

## 📌 Описание
- **Платформа:** Arduino Mega2560 + VexV5 Arduino Shield
- **Моторы:** 2× VexV5 Motor With Encoder
- **Архитектура:** Дифференциальный привод (задний привод)
- **Функции:**
  - Установка скорости левого/правого колеса
  - Интегральная одометрия (`x_mm`, `y_mm`, `heading_deg`)
  - Опциональный PID-регулятор скорости
  - Текстовый UART-протокол с ответами
  - Отладочный вывод по USB

---

## 🔌 Аппаратная схема & Распиновка

| Компонент | Подключение | Примечание |
|-----------|-------------|------------|
| `VEX5_PORT_1` | Правое колесо | Инвертируется программно |
| `VEX5_PORT_5` | Левое колесо | Направление по умолчанию |
| `Serial` (USB) | Мониторинг/отладка | `DEBUG_SERIAL` |
| `Serial2` (TX2=17, RX2=16) | ESP32 / MicroROS | `CMD_SERIAL` |
| `GND` | Общий земляной провод | Обязательно для UART |
| Питание VexV5 | 7.2V–12V внешний источник | Без него моторы уходят в защиту |

## ⚙️ Конфигурация & Калибровка

Перед компиляцией настройте физические параметры робота в начале файла:

```cpp
#define WHEEL_RADIUS_MM   50.0f    // Реальный радиус колеса под нагрузкой [мм]
#define TICKS_PER_REV     1440.0f  // Тиков энкодера на оборот (из доков VexV5)
#define WHEEL_BASE_MM     250.0f   // Расстояние между центрами колёс [мм]
```

🔍 **Метод калибровки:**
1. Прогоните робота по прямой 1000 мм
2. Сравните `POS:x` с рулеткой
3. Подберите `WHEEL_RADIUS_MM` или `TICKS_PER_REV` до погрешности <2%
4. Прогоните поворот на 90°, откалибруйте `WHEEL_BASE_MM`

> 💡 Для переключения между тестом и продакшеном измените макрос:
> ```cpp
> // #define CMD_SERIAL Serial   // Раскомментировать для теста по USB
> #define CMD_SERIAL Serial2     // Раскомментировать для работы с ESP32
> ```

## 📡 Протокол команд (UART)

Все команды отправляются текстом с завершающим символом новой строки (`\n`). В мониторе порта выберите **`Both NL & CR`**.

| Команда | Формат | Ответ | Описание |
|---------|--------|-------|----------|
| `V` | `V\n` | `VER:bro_bot_mega2560/1.1\n` | Версия прошивки |
| `S` | `S:<L>,<R>\n` | `OK\n` / `ERR:SYNTAX\n` | Установить скорость (-1000..1000) |
| `P` | `P\n` | `POS:<x>,<y>,<h>\n` | Запрос одометрии (мм, °) |
| `H` | `H\n` | `HEAD:<deg>\n` | Запрос только курса |
| `E` | `E\n` | `ENC:<L>,<R>\n` | Сырые значения энкодеров |
| `R` | `R\n` | `OK\n` | Сброс одометрии и энкодеров |
| `C` | `C:<tpm>,<wb>\n` | `OK\n` | Калибровка (логирование) |

**Пример сессии:**
```text
V
→ VER:bro_bot_mega2560/1.1

S:300,300
→ OK
# Ждём 2 секунды
S:0,0
→ OK

P
→ POS:587.3,12.1,0.45

R
→ OK
```

## 🛠️ Сборка & Загрузка

### Arduino IDE
1. Откройте `bro_bot_mega2560.ino`
2. Проверьте плату: `Инструменты → Плата: Arduino Mega or Mega 2560`
3. Проверьте порт: `Инструменты → Порт: /dev/ttyUSBx` (Linux) или `COMx` (Windows)
4. Нажмите `✓` (Проверить) → `→` (Загрузить)

### PlatformIO (рекомендуется)
```ini
[env:megaatmega2560]
platform = atmelavr
board = megaatmega2560
framework = arduino
lib_deps = vex-ru/Vex5
monitor_speed = 115200
monitor_filters = send_on_enter
```
```bash
pio run --target upload
pio device monitor
```

## 🐛 Решение проблем

| Симптом | Причина | Решение |
|---------|---------|---------|
| Нет ответа на команды | Команды уходят не на тот порт | Проверьте `#define CMD_SERIAL` (Serial vs Serial2) |
| Зависание на `[INIT]...` | `while(!DEBUG_SERIAL);` ждёт подключения | Нажмите `Enter` в мониторе или закомментируйте строку |
| Моторы гудят, не крутятся | Нет внешнего питания VexV5 | Подключите 7.2V–12V к разъёму питания шилда |
| `ERR:SYNTAX` | Неверный формат или нет `\n` | В мониторе выберите `Both NL & CR`, проверяйте запятые |
| Одометрия "дрейфует" | Проскальзывание колёс / шум энкодера | Включите `USE_PID true`, откалибруйте радиус |
| `Vex5.begin()` падает | Неверный `SPI_NSS` или перемычка | Укажите `Vex5.begin(500000, -1, 0);` согласно шилду |
